# Implementation Plan: Green-Themed Chatbot Widget UI

**Branch**: `main` | **Date**: 2026-02-21 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/2-chatbot-widget-ui/spec.md`

## Summary

Build a frontend-only chatbot widget UI for the DJ College Website using Next.js 15, React, Tailwind CSS, and Framer Motion. The widget provides a conversational interface with a green-themed design (Tailwind Emerald palette), featuring a floating trigger button, animated chat window, message display with distinct user/bot styling, and mobile-responsive behavior.

## Technical Context

**Language/Version**: JavaScript/JSX (ES2022+)
**Primary Dependencies**: Next.js 15.5.12, React 18+, Tailwind CSS 3+, Framer Motion
**Storage**: N/A (frontend-only, no persistence)
**Testing**: Jest + React Testing Library (component tests)
**Target Platform**: Web (desktop and mobile browsers)
**Project Type**: Frontend widget within existing Next.js application
**Performance Goals**:
- Widget load time < 1 second
- Animation completion ≤ 300ms
- Scroll performance 60fps with 50+ messages
- WCAG AA contrast compliance (4.5:1 minimum)
**Constraints**:
- Maximum width 380-420px (desktop)
- Full-screen mode on mobile (≤768px)
- Fixed positioning (bottom: 20px, right: 20px)
- No backend integration in scope
**Scale/Scope**: Single feature widget, ~9 React components

## Phase 0: Research & Discovery

**Status**: ✅ COMPLETE

### Research Topics Investigated

1. **Component Architecture Pattern** → Atomic design with compound components
2. **State Management** → useState + useReducer (no global state needed)
3. **Animation Implementation** → Framer Motion (existing dependency)
4. **Color System** → Tailwind Emerald palette with WCAG AA compliance
5. **Responsive Breakpoints** → Single breakpoint at 768px (mobile-first)
6. **Accessibility** → WCAG AA with semantic HTML + ARIA attributes
7. **Icon System** → Lucide React (existing dependency)
8. **File Organization** → One component per file with barrel exports
9. **Message Data Structure** → Simple object array (id, content, sender, timestamp)
10. **Scroll Behavior** → Native scroll with scrollIntoView()

**Output**: [research.md](./research.md)

**Key Finding**: No new dependencies required. All functionality achievable with existing project dependencies (Next.js 15, React, Tailwind CSS, Framer Motion, Lucide React).
